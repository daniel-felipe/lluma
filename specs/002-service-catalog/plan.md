# Implementation Plan: Service Catalog Management

**Branch**: `002-service-catalog` | **Date**: 2026-04-08 | **Spec**: `specs/002-service-catalog/spec.md`
**Input**: Feature specification from `/specs/002-service-catalog/spec.md`

## Summary

Implement full CRUD for barber services (name, price BRL, duration, active/inactive toggle,
reorder, default templates) using the Actions pattern. On first access (zero services), three
template cards ("Corte", "Barba", "Combo") are shown for quick setup. Reorder via
`@dnd-kit/sortable` with Inertia v3 optimistic auto-save on drag-end. Deleting a service
with future appointments (F4 stub for now) cancels them and sends queued email notifications
to both the affected clients and the barber. Publishing a profile requires ≥ 1 active service.

## Phase 0: Research Summary

See `research.md` for full decision log. Key decisions:

| # | Decision | Outcome |
|---|----------|---------|
| D-001 | Service migration columns | `name`, `price_cents` (unsignedInteger, centavos), `duration_minutes` (smallint), `is_active` (bool), `sort_order` (uint) |
| D-002 | Templates | PHP constants in controller, sent as `templates` Inertia prop when services count = 0 |
| D-003 | DnD library | `@dnd-kit/core` + `@dnd-kit/sortable` (✅ aprovado 2026-04-08) |
| D-004 | Appointments check | `futureAppointmentsCount()` stub returns 0; F4 fills in |
| D-005 | Route structure | Single page `onboarding/services`, EnsureBarberOnboarding updated |
| D-006 | Authorization | Scoped query binding (404 for other barbers' services) |
| D-007 | Price format | `unsignedInteger` centavos in DB, `number` in Inertia props, `Intl.NumberFormat` (÷100) on frontend. Frontend converte input decimal → centavos antes de enviar |
| D-008 | Notification channel | Email via Laravel Notifications + ShouldQueue (client + barber) |
| D-009 | Reorder save | Auto-save on drag-end via `router.patch` + optimistic update |
| D-010 | Validation limits | price_cents max 9999999 (= R$ 99.999,99), duration max 480 min |
